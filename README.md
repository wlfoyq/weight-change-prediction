# Прогноз изменения веса

## Описание задачи

Проект посвящён прогнозированию изменения веса пользователя на основе данных о его питании, физической активности, качестве сна и уровне стресса. Задача — регрессия: предсказать числовое изменение веса за определённый период.

Дополнительно проведён расширенный анализ на большем датасете (UCI) для проверки закономерностей, связанных с влиянием физической активности на вес, на большей выборке.

Финальный результат проекта — не только обученная модель, но и веб-приложение на Streamlit, через которое можно получить прогноз изменения веса (в кг) по введённым параметрам, не запуская код вручную.

## Датасеты

### Основной датасет — регрессия

**Comprehensive Weight Change Prediction** (Kaggle)
https://www.kaggle.com/datasets/abdullah0a/comprehensive-weight-change-prediction

100 записей. Признаки:
- Возраст, пол
- Текущий вес (lbs/kg)
- BMR (базовый обмен веществ)
- Суточная калорийность рациона
- Суточный калорийный дефицит/избыток
- Длительность периода наблюдения (недели)
- Уровень физической активности (Sedentary/Lightly Active/Moderately Active/Very Active)
- Качество сна (Poor/Fair/Good/Excellent)
- Уровень стресса
- **Целевая переменная:** изменение веса (lbs)

### Дополнительный датасет — только для анализа

**Estimation of Obesity Levels Based on Eating Habits and Physical Condition** (UCI ML Repository)
https://archive.ics.uci.edu/dataset/544/estimation+of+obesity+levels+based+on+eating+habits+and+physical+condition

2087 записей (после удаления дубликатов). Данные о пищевых привычках и физической активности жителей Мексики, Перу и Колумбии.

Используется **только для дополнительного EDA-анализа** — проверки закономерностей на большей выборке (например, связь активности и категории веса). Не участвует в обучении итоговой модели, так как решает другую задачу (классификация категории ожирения, а не прогноз числового изменения веса).

## Ограничение

Основной датасет для модели небольшой (100 записей) — результаты стоит рассматривать как предварительные. Для повышения надёжности при обучении моделей использовалась кросс-валидация вместо простого train/test split.

## Использованные признаки (после feature engineering)

- Вес в кг (переведён из lbs)
- Изменение веса в неделю (нормализация по длительности периода)
- Доля калорийного дефицита/избытка относительно BMR

- Закодированные категориальные признаки: пол, уровень активности, качество сна

## Модели и метрики

**Модели**

Baseline — простейшая модель, точка отсчёта для сравнения. Если более сложная модель не превосходит baseline — усложнение не оправдано.
Линейная регрессия ищет зависимость вида y = a₁x₁ + a₂x₂ + ... + b, то есть предполагает, что каждый признак влияет на целевую переменную с постоянным коэффициентом, независимо от значений других признаков. Она не умеет улавливать нелинейные зависимости и взаимодействия между признаками.

Три сложные модели, каждая из которых умеет находить нелинейные зависимости и взаимодействия признаков — то, что недоступно линейной регрессии.

1) **RandomForestRegressor**
Строит много (200 в проекте) независимых деревьев решений, каждое — на случайном подмножестве данных и признаков. Итоговый прогноз — среднее предсказаний всех деревьев. Устойчив к выбросам и переобучению, не требует масштабирования признаков.

2) **XGBRegressor**
Тоже ансамбль деревьев, но строятся они последовательно: каждое следующее дерево старается исправить ошибки предыдущих деревьев. Часто даёт лучшее качество на табличных данных, но чувствительнее к настройке гиперпараметров (learning_rate, глубина деревьев) — легче переобучиться.

3) **KNeighborsRegressor (KNN)**
Не строит явную модель: чтобы предсказать значение для нового объекта, находит k=5 самых похожих объектов в train (по евклидову расстоянию между признаками) и усредняет их целевые значения. Очень чувствителен к масштабированию признаков и плохо работает при малом объёме данных — что и показал этот проект (KNN — самая слабая из трёх сложных моделей).

**Метрики**

Все три метрики отвечают на один вопрос — "насколько сильно модель ошибается", но каждая делает акцент на разном, поэтому их обычно смотрят вместе, а не по одной.

- **MAE** (Mean Absolute Error) — средняя абсолютная ошибка. Показывает, на сколько в среднем модель ошибается, в тех же единицах, что и целевая переменная.
- **RMSE** (Root Mean Squared Error) — корень из средней квадратичной ошибки. Показывает то же самое, что и MAE, но из-за возведения в квадрат сильно штрафует большие ошибки. Если RMSE заметно больше MAE — это сигнал, что у модели есть несколько крупных ошибок, а не равномерно распределённая небольшая погрешность.
- **R²** (коэффициент детерминации) — показывает, какую долю разброса целевой переменной объясняет модель, в диапазоне обычно от 0 до 1 (может быть и отрицательным).

R² = 1 — идеальные предсказания
R² = 0 — модель не лучше, чем просто предсказывать среднее значение по train
R² < 0 — модель хуже, чем наивное среднее (именно это произошло с baseline — 100 строк маловато, и данные шумные)

Не зависит от единиц измерения, поэтому его удобно использовать для сравнения качества моделей на разных задачах.

## Веб-приложение (Streamlit)

Приложение `src/streamlit_app.py` — оболочка над обученными моделями:

- форма ввода: возраст, пол, вес (в **кг**), калорийность рациона, отклонение калорийности от нормы, длительность наблюдения, активность, сон, стресс;
- выбор одной из пяти обученных моделей для прогноза, со сравнением их метрик и предупреждением при выборе baseline (Linear Regression нестабильна на малой выборке — R² < 0 на тесте);
- прогноз выводится **в килограммах** (таргет исходно в фунтах — конвертация на уровне приложения);
- блок сравнения моделей и визуализация важности признаков.

Запуск (из корня репозитория):

```
pip install -r requirements.txt
python src/train.py                       # обучает и сохраняет все модели + scaler в models/
python -m streamlit run src/streamlit_app.py
```

## Выводы

Лучшая модель по итогам всего пайплайна — RandomForestRegressor после подбора гиперпараметров. Она устойчиво (по кросс-валидации) превосходит как линейную регрессию (baseline), так и KNN, и незначительно отличается от GradientBoosting — то есть выбор в пользу ансамблей деревьев для этой задачи оправдан, а перевес в пользу конкретно RandomForest не является решающим и мог бы измениться при большем объёме данных.

Наибольшее влияние на прогнозируемое изменение веса, по данным модели, оказывают уровень стресса и качество сна, а не только прямые "калорийные" показатели (дефицит калорий, потребление калорий). Это содержательный вывод, согласующийся с современными представлениями о влиянии психологических и поведенческих факторов на вес, но с оговоркой: при 100 наблюдениях эту связь стоит рассматривать как гипотезу для дальнейшей проверки, а не как окончательно установленный факт.

Главное ограничение проекта — размер датасета. Оно напрямую повлияло на методологические решения (кросс-валидация вместо одиночного split, умеренные сетки гиперпараметров, осторожная формулировка выводов) и должно быть явно указано при презентации результатов. Возможное направление развития проекта — сбор дополнительных данных по основной задаче и, отдельно, доведение до готового результата второй, классификационной задачи (категория ожирения) на датасете UCI, который был сознательно отложен на этом этапе, а также расширение веб-приложения (например, сохранение истории прогнозов).
